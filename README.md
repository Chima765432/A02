## Part 1: Tutorial - How to Use Git, WebStorm, and GitHub

### Setting Up Git

1. Go to https://git-scm.com/downloads and download Git for your operating system.
2. Run the installer and follow the default settings.
3. Once installed, open your terminal or command prompt and type `git --version` to confirm it installed correctly.

### Setting Up WebStorm

1. Go to https://www.jetbrains.com/webstorm/download and download WebStorm.
2. Run the installer and follow the setup steps.
3. Open WebStorm once installation is complete.
4. Go to **File > Settings > Version Control > Git** and make sure WebStorm detects your Git installation.

### Connecting WebStorm to GitHub

1. In WebStorm, go to **File > Settings > Version Control > GitHub**.
2. Click **Add Account** and log in with your GitHub credentials.
3. Click **Apply**, then **OK**.

### Cloning a GitHub Repository into WebStorm

1. In WebStorm, go to **File > New > Project from Version Control**.
2. Select **GitHub** and choose the repository you want to clone.
3. Choose a local folder to save it in and click **Clone**.

### Making Changes and Committing

1. Open a file in your project and make your changes.
2. Go to **Git > Commit** in the top menu.
3. Write a clear commit message, for example: `Task: Create Repository`.
4. Click **Commit**.

### Pushing Changes to GitHub

1. After committing, go to **Git > Push**.
2. Click **Push** to send your changes to GitHub.

 ## Part 2: Glossary

- **Branch**: A separate version of the repository that lets you work on changes without affecting the main code.
- **Clone**: A copy of a remote repository downloaded to your local machine.
- **Commit**: A saved snapshot of your changes in the repository, along with a message describing what was changed.
- **Fetch**: The act of downloading changes from a remote repository without merging them into your current work.
- **GIT**: A free, open-source version control system that tracks changes in files and allows multiple people to work on a project together.
- **Github**: A web-based platform that hosts Git repositories and allows collaboration on code.
- **Merge**: The process of combining changes from one branch into another.
- **Merge Conflict**: An error that occurs when two branches have changes in the same part of a file and Git cannot automatically decide which version to keep.
- **Push**: Sending your committed changes from your local machine to a remote repository.
- **Pull**: Downloading and merging changes from a remote repository into your local copy.
- **Remote**: A version of your repository stored on a server, such as GitHub, rather than on your local machine.
- **Repository**: A storage location where your project files and the full history of changes are kept.
